Okta
UI Engineer on the MFA/Authentication Team (Summer 2016)
Always On

Okta is the foundation for secure connections between people and technology. To that end, Okta is "Always On": Okta not only has astounding uptime, but a company culture of being dialed-in, focused, and excited to contribute every day. During Summer 2016, I had the pleasure of working on the MFA/Authentication team, which implements methods/factors of authentication into Okta, policy rules and enforcement, third-party authentication integrations, and data-analytics-driven access control flows.

UI Engineering @ Okta

As an intern on the MFA/Auth team, I had the pleasure of working with unbelievably talented and experienced engineers as a UI engineer. Coming into the internship I had very limited experience with frontend development, so it was a great experience jumping right into it and growing in that area. The core UI team has created an in-house UI framework called Courage, built on Backbone.js, with Require.js for dependency management, which we leverage to build the frontend components for new features. I contributed to Courage as well, adding addional Playground demos, fixing bugs on existing features, and adding small additional features where convenient. During my internship, my main project was the completion of the UI for the SMS Localization feature, for which I migrated incomplete legacy JSP code onto the Courage framework, updated the existing UI, and added support for translations of SMS notification templates.

Okta 2016 Intern Hackathon: We Won!

Every year, Okta holds an intern hackathon! After a lot of hard work from every team, and a great deal of hacking over the weekend, my team got first place (and nifty LIFX IoT lightbulbs to boot!) with our project: Suspicious User Detection System. As the sole UI engineer on my team, I built the entirety of the frontend for our hackathon project, which was also where most of code and logic lived for our project. In order to be able to incorporate our hack idea more seamlessly into the product, we built everything off of the existing codebase, and I used Courage to build the frontend. Events examined for suspicious activity were returned via queries against a database using the in-house Sage API. The final visual result was the integration of a pop-up modal notification directly into the current single sign-on main page for endusers. Shoutouts to Joseph, Jason, and Kenton for being awesome team members. Always Believe.

Web Design Decal
Excited Student (Fall 2016)
Confidence to Code, Design, and Create

UC Berkeley's Web Design Decal is a great DeCal (class run by students, for students) taught each semester with lots of love and caffeine. It's a tremendous class taught by enormously talented instructors, and I was impressed bigly. Admission is extremely competitive, as instruction both on the design and on the coding and is both impeccable, and students come out with a one-of-a-kind introduction to frontend web development.

Woohoo!

As a Fall 2016 student of the Web Design Decal, I had lots of fun learning design principles, which really can make or break a site. As the saying goes, as a developer: if you have a good backend, good, that's your job; if you have a good frontend, great, people are actually going to want to use your site because it's functional, easy-to-use, and beautiful. Granted, scalable systems, data integrity, and uptime are all critical as well, but I'm just being tongue-in-cheek on this one. In any event, it was also great opportunity to fill in all of my gaps of knowledge regarding the fundamentals of HTML, CSS, and jQuery as well.

Made With Love in Berkeley

This site was made as my final project for the Web Design Decal, and it truly was a great chance to demonstrate some of the techniques and patterns I've picked up. Coming into college, a lot of students have disdain for frontend development for some reason (perhaps because it's different from academic CS, and doesn't necessarily involve abstruse algorithms everywhere), but as a senior leaving Berkeley, I came to appreciate even more the principles of design and frontend programming. It genuinely into the "both a science and an art" bucket, and I'm glad I got to experience the class. Thanks for teaching this the Web Design Decal.
